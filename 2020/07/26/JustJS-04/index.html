简介

Dan Abramov:Redux作者，React社区大神。
前一段时间发起了一个活动，通过邮箱订阅他写的一个小专栏，每写好一片草稿就会发到你的邮箱里面。
这是这篇小专栏的第四节：‘ Counting the Values(Part 1) ’–“计算值(第一部分)”。
下面内容是在原作基础上，用我的大白话翻译的文章内容，文章原文贴至末尾。

在本模块中，我们将进一步了解JavaScript的世界及其贯穿它世界中的值。但在我们做到这一点之前，我们需要解决房间里的大象。JavaScript的世界是真实的吗？

JavaScript模拟(The JavaScript Simulation)

我生活在JavaScript宇宙中的小行星上。

每当我’询问’JS一个问题的时候，它总是反馈我一个值。当然所有这些值都不是我想象出来的。变量、线、值–他们充斥在我世界的周围。我周围的JavaScript世界对我来说是绝对真实的，就像你生活的世界对你来说是真实的一样。

但有时，在下一行代码执行之前会有片刻的间隙。像下一个函数调用之前的空闲时间–好似矩阵里的一个小故障。在那些时刻，我看到了一个比我的大得多的世界。

当世界在我眼前显现的时候，它没有所谓变数和值，没有表达式，没有文字。相反，有夸克，原子，电子，水和生命。也许，您也是以这种方式对这个世界熟悉的？

那个称为“人类”的生灵使用称为“计算机”的特殊机器来模拟我的 JavaScript宇宙。他们中有些人是出于娱乐目的。他们中有些人是牟利的。他们中有些人根本没有理由这样做。他们一时兴起，整个世界每天出生并死亡一万亿次。
毕竟，也许我的JavaScript世界不是那么真实。
这意味着有两种研究方法。

从外部进行学习(Studying From the Outside)

研究我的JavaScript世界的一个方法就是从外部去研究它。
也许，您可能会专注于“真实”地模拟我的世界（一个JavaScript引擎）的工作方式。例如，您可能会了解到，这个文本字符串（ 我的世界中的一个值）是存储在硅芯片中的字节序列。
这种方法使我们将精力集中在人和计算机的物理世界上。一些教程采用这种方法。但是我的方法是不同的。

从内部进行学习(Studying From The Inside)

我们将从内部研究JavaScript世界。现在想象你跟我一起进入了JavaScript世界，并站在我旁边。我们将观察JS宇宙的定律并进行实验，就像物理学家在物理宇宙中所做的那样。

我们将了解JavaScript世界的本质，而无需考虑其实现方式。这类似于物理学家如何谈论恒星的性质而无需回答物理世界是否真实的问题。这都不是问题！我们绝逼可以用自己的术语来描述它。

我们的思维模式构建过程中并不会尝试回答诸如“计算机内存中的值如何表示？”之类的问题。这样的答案一直在变化！实际上，即使您的程序正在运行 ，该问题的答案也会改变。如果您听到有关JavaScript“真正”如何表示内存中的数字，字符串或对象的简单说明，则很可能是错误的。

对我来说，每个字符串都是一个值。不是“指针”或“内存地址”，而是一个值 。 在我的宇宙中，值就足够了。不要让“内存单元”和其他低级隐喻使您分心以建立准确的JavaScript高级别思维模型。反正对于我们来说，理解起来都是乌龟在爬！

如果您使用的是较低级的语言，请保留关于“通过引用传递”，“在堆栈上分配”，“在写时复制”等等的直觉。这些关于计算机工作方式的模型通常使人们更难以确定JavaScript程序中可能发生或不会发生的事情。我们将研究一些较低级别的细节，但仅在真正重要的地方 。它们可以作为我们心理模型的补充 ，而不是其基础。

相反，我们思维模式的基础是我们的世界充满值。每个值都属于几种内置类型之一。其中一些是原始的，这使得这些类型的值不可变。变量是从代码中的名称指向值的“线”。而且，我们将继续在此基础上发展。

至于一些奇怪的愿景，我不再对它们抱有太多的想法。我有要指出的问题，要问的问题和要调用的函数。我最好马上去做！

当我看着它们时，星星是明亮的。

我眨眼时他们还在吗？

我耸耸肩。

“一探究竟去”

计算值(Counting the Values)

弗恩·伯爵（Count von Count）是我童年的榜样。如果您对芝麻街不熟悉，那你不了解他最喜欢的消遣就是数数。今天，弗恩·伯爵（Count von Count）将与我们一起数数JavaScript领域中的每个值。

您可能想问： 数数值有什么意义？我们又不是上算术课。计数的本质是将事物彼此区分开。当您清楚地看到它们是两个不同的苹果时，您只能说有“两个苹果”。相互区分值是理解JavaScript 相等性的关键-这将是我们的下一个主题。

就像维吉尔（Virgil）引导但丁（Dante）穿越地狱的9个循环一样，弗恩·伯爵（Count von Count）将陪伴我们穿越JavaScript的“星体”，去了解他的一些值：布尔值，数字，字符串等等。就让我当成是一次观光旅游吧。

你会发现，JavaScript无法实现我们在现实生活中使用的那种数学。浮点数学是“计算机数学”。不用太担心这个名称或它如何工作。很少有人知道它的所有微妙之处，这就是重点！它在实践中效果很好，大多数时候您都不会考虑它。不过，让我们快速看一下它与众不同的原因。

颜色和数字(Colors and Numbers)

您是否曾经使用扫描仪将实物照片或文档转换为数字文档？这个类比可以帮助我们理解JavaScript的Number。

扫描仪通常最多可分辨1600万种颜色。如果您用红色蜡笔绘制图片并进行扫描，则扫描后的图像也会发出红色-但它将具有扫描仪从这1600万种颜色中挑选出的最接近的红色。因此，如果您有两个颜色略有不同的红色蜡笔，扫描仪可能会以为它们的颜色完全相同！

可以说，扫描仪将颜色视为精度有限。

浮点数学是相似的。在实际数学中，存在无限数量的数字。但是在浮点数学中， 只有18百万个五位数。因此，当我们在代码中编写数字或进行数字计算时，JavaScript会选择它知道的最接近的数字，就像我们的扫描仪对颜色一样。

换句话说，JavaScript将数字视为具有有限的精度。

幸运的是， Number.MIN_SAFE_INTEGER和Number.MAX_SAFE_INTEGER之间的任何整数都是准确的。这就是为什么10 + 20 === 30 。

但是当我们写0.1或0.2 ，我们得不到精确的 0.1和0.2 。我们获得JavaScript中最接近的可用数字。它们几乎完全相同，但是可能会有微小的差异。这些微小的差异加起来，这就是为什么0.1 + 0.2不能提供与编写0.3 完全相同的数字的原因。

如果这仍然令人困惑，请不要担心。您可以了解有关浮点数学的更多信息 ，但您已经比开始编写本指南时了解的更多！除非您使用财务应用程序，否则您不必担心这一点。

使用这些特殊数字编写代码并不常见。但是，它们可能由于编码错误而出现。因此，知道它们的存在也是令人欢乐的。

未完待续(To be Continuted)

该模块分为两部分。我们已经到了第1部分的结尾。我们现在休息一会儿。让我们回顾一下到目前为止我们已经数了多少个值！

  • Undefined ：只有一个值， undefined 。
  • Null ：仅一个值， null 。

  • Numbers ：每个浮点数学数字一个值。

我们还了解了一些有关JavaScript数字的有趣事实：

  • 并非所有数字都可以用JavaScript完美地表示。它们的小数部分提供接近0更高精度，而远离它的精度更低。我们可以说它们的小数点是“浮动”的。
  • 无效的数学运算（例如1 / 0或0 / 0是特殊的。 NaN就是这样的数字之一。它们可能是由于编码错误而出现的。
  • typeof(NaN)是一个数字，因为NaN是一个数值。这就是所谓的“非数”，因为它代表了一个“无效”号的主意 。
